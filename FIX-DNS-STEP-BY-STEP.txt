================================================================================
                    FIX DNS TO CONNECT TO MONGODB ATLAS
================================================================================

PROBLEM: Your computer's DNS cannot find MongoDB Atlas servers
SOLUTION: Change DNS to Google DNS (8.8.8.8)

================================================================================
                              STEP-BY-STEP GUIDE
================================================================================

STEP 1: Open Network Connections
---------------------------------
1. Press Windows Key + R on your keyboard
2. Type: ncpa.cpl
3. Press Enter

You will see a window with your network connections.


STEP 2: Open Network Properties
--------------------------------
1. Find your ACTIVE network connection (the one you're using now)
   - It might be called "Wi-Fi" or "Ethernet"
   - It will have a colored icon (not grayed out)

2. RIGHT-CLICK on your active connection
3. Click "Properties"

A new window will open.


STEP 3: Configure IPv4 Settings
--------------------------------
1. In the list, find "Internet Protocol Version 4 (TCP/IPv4)"
2. Click on it to select it (single click)
3. Click the "Properties" button

A new window will open.


STEP 4: Change DNS Servers
---------------------------
1. You will see two options:
   - "Obtain DNS server address automatically" (probably selected)
   - "Use the following DNS server addresses"

2. Click on "Use the following DNS server addresses"

3. In the boxes that appear, enter:
   
   Preferred DNS server:  8.8.8.8
   Alternate DNS server:  8.8.4.4

4. Click "OK"
5. Click "OK" again on the previous window
6. Close all windows


STEP 5: Flush DNS Cache
------------------------
1. Press Windows Key + R
2. Type: cmd
3. Press Enter

4. In the black window that opens, type:
   ipconfig /flushdns

5. Press Enter
6. You should see: "Successfully flushed the DNS Resolver Cache"
7. Close the window


STEP 6: Restart Your Computer
------------------------------
1. Save all your work
2. Close all programs
3. Restart your computer

This is IMPORTANT! The DNS change needs a restart to take effect.


STEP 7: Test DNS After Restart
-------------------------------
After your computer restarts:

1. Open Command Prompt or PowerShell
2. Navigate to your project:
   cd D:\connectnow\-ConnectNow

3. Run the test:
   node test-atlas-connection.js

4. Look for the result:
   ✅ "DNS is working!" = SUCCESS! MongoDB Atlas will connect
   ❌ "SRV Resolution Failed" = DNS issue still exists


STEP 8: Restart Backend (If DNS Works)
---------------------------------------
If the test shows "DNS is working!":

1. Stop your backend server (if running)
2. Start it again:
   cd backend
   npm run dev

3. You should see:
   "✅ SUCCESS! Connected to MongoDB Atlas"

4. Check MongoDB Atlas dashboard - you should see your data!


================================================================================
                              ALTERNATIVE SOLUTION
================================================================================

If changing DNS doesn't work or you can't do it:

OPTION A: Use Mobile Hotspot
-----------------------------
1. Enable hotspot on your phone
2. Connect your computer to the hotspot
3. Run: node test-atlas-connection.js
4. If successful, restart backend
5. MongoDB Atlas will connect!


OPTION B: Deploy to Render (BEST FOR PRODUCTION)
-------------------------------------------------
1. See: RENDER-DEPLOYMENT-GUIDE.md
2. Deploy your app to Render cloud
3. MongoDB Atlas works automatically
4. No DNS issues on Render servers
5. Your app becomes publicly accessible


================================================================================
                                  SUMMARY
================================================================================

What to do:
1. Change DNS to 8.8.8.8 and 8.8.4.4
2. Restart computer
3. Test: node test-atlas-connection.js
4. If successful, restart backend
5. Check MongoDB Atlas dashboard

Why this works:
- Google DNS (8.8.8.8) can resolve MongoDB Atlas servers
- Your current DNS cannot
- This is a common issue with ISP DNS servers

What if it doesn't work:
- Use mobile hotspot (quick test)
- Deploy to Render (best for production)
- Keep using local MongoDB (works perfectly for development)
